## Common Interview Questions & Answers

### Q: "Walk me through your project"
**A:** "I analyzed 7,000 telecom customers to understand churn. First, I explored patterns in contract type, tenure, and services. Key finding: month-to-month contracts have 42% churn vs 3% for 2-year contracts. I then built a logistic regression model to predict churn with 81% accuracy. This enables the business to target retention efforts, potentially saving $2.5M+ annually."

### Q: "What was the biggest challenge?"
**A:** "The biggest challenge was understanding feature relationships. At first, I thought tenure and monthly charges were the key drivers, but analysis showed contract type was far more important. This taught me the importance of thorough EDA before jumping to conclusions."

### Q: "How would you improve this project?"
**A:** "Good question! Next steps would be:
1. Try ensemble methods (Random Forest, XGBoost) for better predictions
2. Perform deeper cohort analysis by geography and service type
3. Build an automated monitoring dashboard for real-time churn scoring
4. Conduct A/B testing on retention campaigns
5. Incorporate customer feedback/sentiment data"

### Q: "How do you handle imbalanced data?"
**A:** "Great question! Churn is imbalanced (26% positive class). In this project, I focused on precision-recall and ROC-AUC rather than accuracy. In production, I'd consider:
- Stratified train-test split (which I used)
- SMOTE oversampling
- Class weight adjustment in the model
- Threshold tuning based on business cost"

### Q: "How would you deploy this model?"
**A:** "I'd build an API endpoint using Flask/FastAPI that:
1. Takes customer features as input
2. Returns churn probability score
3. Logs predictions for model monitoring
4. Updates daily with new customer data
5. Triggers alerts for high-risk customers"
